Guía de Uso - API y Utils

1. AUTENTICACIÓN

Registrar Nuevo Usuario

// En auth/registro.html, el formulario hace esto:
const respuesta = await registrar({
    nombre: "Juan Pérez",
    email: "juan@email.com",
    telefono: "+57 300 123 4567",
    password: "MiPassword123",
    rol: "DONADOR"
});

// Si es exitoso, redirige a login
// El backend debe crear el Usuario en BD

Login

// En auth/login.html, hace esto:
const respuesta = await login("juan@email.com", "MiPassword123");

// Respuesta esperada:
{
    token: "eyJhbGciOiJIUzUxMiIs...",
    rol: "DONADOR",
    id: 1
}

// Se guarda automáticamente en localStorage
localStorage.getItem('token');  // Token JWT
localStorage.getItem('rol');    // DONADOR | BENEFICIARIO | ADMIN

Verificar Autenticación

// Usar en cualquier página
if (estaAutenticado()) {
    console.log("Usuario autenticado");
    console.log("Rol:", obtenerRol());
} else {
    window.location.href = '/login';
}
2. OPERACIONES CON DONACIONES

Obtener Todas las Donaciones

// Usado en landing.html para llenar el mapa
const donaciones = await getDonaciones();

// Respuesta (array):
[
    {
        id: 1,
        tipo: "comida",
        descripcion: "Arroz de 5kg",
        lat: 2.941761,
        lng: -75.297825,
        estado: "DISPONIBLE"
    },
    // ... más donaciones
]

Crear Nueva Donación (Donador)

// Proteger: solo DONADOR
protegerRuta('DONADOR');

// Crear donación
const nueva = await crearDonacion({
    tipo: "ropa",
    descripcion: "Camisetas manga larga talla M",
    categoria: "ropa",
    lat: 2.941761,
    lng: -75.297825,
    fotoUrl: "https://ejemplo.com/foto.jpg"
});

console.log("Donación creada:", nueva.id);

Cambiar Estado de Donación

// Donador: DISPONIBLE → RESERVADO → ENTREGADO
await cambiarEstadoDonacion(1, "RESERVADO");
await cambiarEstadoDonacion(1, "ENTREGADO");

// El beneficiario ve esto reflejado en el mapa

Obtener Donaciones del Usuario

// En dashboard del donador
const usuarioId = localStorage.getItem('usuario_id'); // Del login
const misDonaciones = await getDonacionesPorUsuario(usuarioId);

misDonaciones.forEach(donacion => {
    console.log(`${donacion.tipo}: ${donacion.estado}`);
});
3. SOLICITUDES DE RECOGIDA

Beneficiario: Crear Solicitud

// Proteger: solo BENEFICIARIO
protegerRuta('BENEFICIARIO');

// Solicitar una donación
const solicitud = await crearSolicitud({
    beneficiarioId: 5,  // ID del beneficiario
    donacionId: 1,      // ID de la donación
    notas: "¿Puedo recoger el viernes?"
});

console.log("Solicitud creada:", solicitud.id);
console.log("Estado:", solicitud.estado); // PENDIENTE

Ver Solicitudes Recibidas (Donador)

// El donador ve solicitudes de sus donaciones
const donadorId = 3;
const solicitudes = await getSolicitudes(donadorId);

solicitudes.forEach(s => {
    console.log(`Beneficiario: ${s.beneficiario.nombre}`);
    console.log(`Estado: ${s.estado}`); // PENDIENTE, ACEPTADA, ENTREGADA
});

Actualizar Estado de Solicitud

// Donador acepta o rechaza
await cambiarEstadoSolicitud(1, "ACEPTADA");
// O
await cambiarEstadoSolicitud(1, "RECHAZADA");

// Cuando es ENTREGADA, la donación cambia a ENTREGADO
4. UTILIDADES Y COMPONENTES

Notificaciones

// Mostrar notificación (desaparece en 5s)
mostrarNotificacion('¡Donación creada!', 'success');

// Otros tipos
mostrarNotificacion('Error al enviar', 'error', 3000);
mostrarNotificacion('Operación completada', 'success', 2000);
mostrarNotificacion('Advertencia', 'warning', 5000);
mostrarNotificacion('Información', 'info', 0); // No desaparece

Modales de Confirmación

// Pedir confirmación antes de eliminar
confirmar(
    '¿Eliminar donación?',
    'Esta acción no se puede deshacer',
    async () => {
        await eliminarDonacion(1);
        mostrarNotificacion('Eliminada', 'success');
    },
    'Sí, eliminar'
);

Formateo de Fechas

// Múltiples formatos
formatearFecha(new Date(), 'DD/MM/YYYY');
// → "15/04/2026"

formatearFecha(new Date(), 'DD/MM/YYYY HH:MM');
// → "15/04/2026 14:30"

formatearFecha(new Date(), 'YYYY-MM-DD');
// → "2026-04-15"

tiempoTranscurrido(new Date(Date.now() - 3600000));
// → "Hace 1 hora"

6. FLUJO COMPLETO DE USUARIO

1. Usuario Anónimo

Accede a "/" → ve Landing → puede ver mapa y donaciones

2. Se Registra como DONADOR

POST /api/auth/registro → Token guardado → Redirige a /donador/dashboard

3. Donador Crea Donación

POST /api/donaciones → Se muestra en el mapa → Estado DISPONIBLE

4. Beneficiario Solicita

POST /api/solicitudes → Notificación al donador → Estado PENDIENTE

5. Donador Acepta

PATCH /api/solicitudes/{id}/estado → ACEPTADA → Donación → RESERVADO

6. Coordinan Entrega

Mensajes, teléfono → Se reúnen → Intercambio

7. Completar Solicitud

PATCH /api/solicitudes/{id}/estado → ENTREGADA → Donación → ENTREGADO
